# Output Contracts

## 1. Workflow Design Contract
When the skill is used to design a workflow, the output should include:
- Workflow Goal
- Five-Layer Architecture
- Recommended MVP
- Scale-Up Path
- Immediate Next Action

## 2. Workflow Execution Contract
When the skill is used to actually run a content workflow, the output should preferably generate these artifacts:
- `source.md` or equivalent normalized source
- `summary.md` or key-point extraction
- one long-form asset
- one short-form asset
- optional visual asset prompt or generated image
- deposition record

## 3. Distribution Contract
When distribution is involved, the skill should say clearly:
- which channels were targeted
- whether the result is a draft or published output
- what still requires human review

## 4. Deposition Contract
A deposition should contain at least:
- source
- summary
- key points
- outputs created
- storage location
- next possible reuse direction
